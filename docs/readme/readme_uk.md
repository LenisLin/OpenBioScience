## Головна лінія

OpenScience спирається на ключову ідею Claude Science: науковий AI не має бути лише чатом. Це має бути дослідницьке середовище для проєктів, аналізу, пошуку джерел, наукових artifacts, перегляду файлів, обчислювальних журналів і перевірки результатів.

| Дослідницьке питання | Відповідь OpenScience |
|---|---|
| Де живе робота | У папці дослідницького проєкту, а не лише в розмові |
| Чи можна запускати реальний аналіз | Так, через Python, R, shell, notebooks і локальні coding agents |
| Чи можна перевірити результат пізніше | Так, фігури, таблиці, notebooks, звіти й рукописи відкриваються як artifacts із слід джерел |
| Як працювати з медичними доказами | Medical Evidence Mode створює звіти із силою доказів, конфліктами та обмеженнями |
| Чи можна використовувати наявні інструменти | Так, локальні файли, скрипти, провайдери моделей і coding-agent workflows |

## Дослідницький процес

| Крок | Що робить дослідник | Що зберігає OpenScience |
|---:|---|---|
| 1 | Створює або відкриває проєкт | Папка, налаштування, джерела, результати |
| 2 | Ставить питання природною мовою | Завдання, припущення, файли, уточнення |
| 3 | Шукає та читає докази | Мітки для статей, випробувань, документів, даних, фігур або запусків коду |
| 4 | Запускає аналіз | Скрипти, команди, notebooks, входи, журнали, середовище |
| 5 | Перевіряє artifacts | Фігури, таблиці, звіти, рукописи, слід джерел, review |
| 6 | Редагує й експортує | Версії, коментарі, PDF, Word, LaTeX, notebooks |
