## Главная линия

OpenScience следует ключевой идее Claude Science: научный AI не должен быть просто чатом. Это должна быть исследовательская среда для проектов, анализа, поиска источников, научных artifacts, предпросмотра файлов, вычислительных журналов и проверки результатов.

| Исследовательский вопрос | Ответ OpenScience |
|---|---|
| Где живет работа | В папке исследовательского проекта, а не только в переписке |
| Можно ли запускать реальный анализ | Да, через Python, R, shell, notebooks и локальные coding agents |
| Можно ли проверить результат позже | Да, фигуры, таблицы, notebooks, отчеты и рукописи открываются как artifacts с цепочку источников |
| Что с медицинскими доказательствами | Medical Evidence Mode строит отчеты с силой доказательств, конфликтами и ограничениями |
| Можно ли использовать существующие инструменты | Да, локальные файлы, скрипты, провайдеры моделей и coding-agent workflows |

## Исследовательский процесс

| Шаг | Что делает исследователь | Что сохраняет OpenScience |
|---:|---|---|
| 1 | Создает или открывает проект | Папка, настройки, источники, результаты |
| 2 | Задает вопрос естественным языком | Задача, допущения, файлы, уточнения |
| 3 | Ищет и читает доказательства | Метки для статей, испытаний, документов, данных, фигур или запусков кода |
| 4 | Запускает анализ | Скрипты, команды, notebooks, входы, журналы, окружение |
| 5 | Проверяет artifacts | Фигуры, таблицы, отчеты, рукописи, цепочку источников, review |
| 6 | Исправляет и экспортирует | Версии, комментарии, PDF, Word, LaTeX, notebooks |

## Охват источников

| Источники | Использование |
|---|---|
| 11M+ papers | Обзоры литературы, сравнение методов, цитируемое письмо |
| 225K+ документов по препаратам и устройствам | Метки, руководства, регуляторный контекст, безопасность |
| 1M+ клинических испытаний | Вмешательства, исходы, статус, сравнение, критерии |
| 150M+ abstracts | Быстрый поиск релевантных работ |
| Локальные файлы и outputs | Данные, код, фигуры, notebooks, отчеты, журналы |
