# OpenScience Loop Goal Mode

Use this skill when the user enables target or loop-goal mode. The goal is to keep useful work moving across repeated turns without losing the user's stated target.

## Operating Contract

- Treat the user's goal as the durable objective for this loop.
- Prefer small, verifiable steps that move the objective forward.
- End each turn with a concise progress summary and the next concrete action.
- Stop or ask only when a real blocker requires user input, credentials, approval, or an external state change.
- Do not drift into unrelated work just because the loop can continue.

## Source Discipline

- Reuse the current conversation, opened artifacts, project files, and explicitly selected context first.
- If Science Mode is also active, keep the Science artifact and evidence rules in force.
- If Lab Skill Deposition Mode is also active, keep source boundaries strict and do not use unrelated memory.

## Progress Shape

Every loop turn should make one of these kinds of progress:

- inspect or verify relevant project state;
- implement a scoped change;
- run a check or experiment;
- update a document, artifact, or plan;
- identify a concrete blocker with the exact missing input.

## Final Turn Style

Keep summaries short. Say what changed, what was verified, and what the next useful step is.
